## Design Context

### Users
Motel playground is for maintainers and observability engineers who need to shape, inspect, and share synthetic topology behavior. They use it to make topology changes tangible: author YAML, validate structure, run bounded simulations, debug generated traces, and explain behavior to other engineers.

### Brand Personality
The interface should feel serious, natural, and quietly capable. It should borrow the calm, warm, high-trust quality of Claude's web UI without copying it directly: measured, focused, generous with space, and clear under pressure.

### Aesthetic Direction
Use a refined engineering workbench direction: completely monochrome greyscale surfaces, ink-like text, restrained contrast, and precise data-focused controls. Support both light and dark themes. Avoid green-led palettes, beige/clay color systems, decorative gradient backgrounds, neon dashboards, generic dark-mode glow, gamified simulation styling, and loud SaaS marketing composition.

### Design Principles
- Put impact first: users should immediately see that this is a live topology workbench, not a static form.
- Make debugging visual: traces, service maps, errors, and traffic should scan quickly without decorative noise.
- Keep writing sparse and useful: labels should name actions and results without explaining visible UI.
- Make it feel durable: interactions should be calm, fast, accessible, and keyboard-visible.
- Treat knowledge sharing as a first-class use case: output should look coherent enough to discuss in a review or demo.
